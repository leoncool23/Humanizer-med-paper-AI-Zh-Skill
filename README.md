# Humanizer-med-paper-AI-Zh-Skill



> 专为 AI Cli工具（Claude Code、OpenClaw、Codex 等）设计的中文医学论文降AI率的技能（Skill）

本项目提供了一个高度专业化的 Skill 配置文件，旨在帮助医学研究者、科研人员将AI生成的中文医学学术论文、临床研究报告和系统综述，转化为**资深研究者写作行文风格**，同时确保临床数据、统计量和学术规范的绝对安全。

## ✨ 核心特性

- 🧠 **两轮递进改写**：拒绝一步到位导致的逻辑断裂。第一轮词汇句法层重构，第二轮专项清除学术 AI 模板痕迹。
- 🛡️ **极致数据保真**：硬性约束机制，绝对不改动 P值、OR值、95%CI、文献编号、药物剂量及检验指标单位。
- 🔍 **精准靶向清除**：专门针对 AI 写作的重灾区（如泛化文献归因、意义夸大、模板式局限性段落、空洞展望套话）进行打击。
- 🎯 **智能意图识别**：内置触发词匹配机制，输入特定指令即可无缝接入工作流。

## 📂 适用文档范围

- 医学期刊原创论文、病例报告、临床研究摘要
- 系统综述、Meta 分析、叙述性综述
- 学位论文医学章节、住院医师科研报告
- 临床试验注册材料、伦理审查申请书

## ⚙️ 工作原理

本 Skill 采用严格的顺序执行机制，单次调用仅执行一轮，防止模型“越级”导致文本崩坏：

### 第 1 轮：词汇句法层人性化改写
将“机器报告风格”转为“人类思考落笔风格”。通过动词短语扩展（如“证实”→“得到证实”）、系统性词汇替换、句法冗余化重构（增加合理的学术限定语）以及逻辑连接词松散化，打破 AI 文本特有的机械节奏。各节段（摘要、引言、方法等）采用不同力度的改写策略。

### 第 2 轮：学术论文 AI 模式检测与清除
在第 1 轮基础上，进行深度“排雷”：
1. **具名化归因**：将“研究表明”强制绑定具体文献或删除。
2. **收缩意义夸大**：将“填补了领域空白”替换为具体的知识增量。
3. **拆解模板式局限**：删除“本研究存在以下局限性”等引导句，直接陈述与本设计挂钩的具体缺陷。
4. **压缩展望套话**：删除“未来需要更多研究”，保留有具体人群/终点指标的建议。
5. **消除高频词滥用**：清除“此外”“值得注意的是”“深入探讨”等 AI 标志性词汇。

## 🚀 如何使用

### 1. 安装配置
将本仓库中的 `SKILL.md` 文件放置到你的 AI 助手 Skill 读取目录下：

- **Claude Code**: 通常放置于项目根目录的 `.claude/skills/` 或直接放在工作区根目录。
- **OpenClaw**: 放置于 `.claw/skills/` 或根据具体配置的 `skill_path` 映射。
- **Codex / 其他兼容工具**: 放置于智能体能够读取上下文的 `./skills/` 或 `./prompts/` 文件夹中。

### 2. 触发 Skill
在对话中输入以下任意关键词，智能体将自动加载此 Skill 并**默认执行第 1 轮**：

- `医学论文降AI`
- `期刊投稿降AIGC`
- `研究摘要去AI味`
- `临床研究报告降AI率`
- `综述文章人性化改写`
- `SCI论文降AIGC检测`

### 3. 两轮交互流程
由于安全机制限制，Skill 不会一次性输出最终结果，请按以下步骤操作：
1. **输入文本 + 触发词** ➡️ 获得第 1 轮改写结果。
2. **在同一对话中回复**：“继续执行第2轮” ➡️ 获得最终清除 AI 痕迹的版本。

## 🛑 安全与约束声明

- **零幻觉原则**：Skill 严格禁止新增任何事实、数据、文献引用、实验结论或临床判断。
- **方向锁定**：不得改变原文对疗效、预后或风险的判断方向。
- **语体守卫**：全程保留学术被动语态，绝不口语化。
- **纯净输出**：不输出任何说明性文字、润色建议或 Markdown 粗体（除非原文有）。
